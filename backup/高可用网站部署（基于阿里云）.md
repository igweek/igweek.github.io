## **一、企业背景（真实场景设定）**

某中型电商公司在业务增长后，原有单机部署的网站频繁出现：

* 高峰期访问卡顿甚至宕机
* 数据库压力过大
* 无法应对突发流量（如促销活动）
* 数据安全性不足

公司决定将系统迁移到 阿里云，构建一个**高可用、可扩展、安全的云架构**。

---

## **二、实验目标**

学生需要完成：

1. 搭建云上网络环境（VPC）
2. 部署Web服务器（ECS + Nginx + WordPress）
3. 使用云数据库（RDS MySQL）
4. 配置负载均衡（SLB）
5. 实现高可用架构
6. 验证系统访问与容灾能力

---

## **三、整体架构（理解重点）**

```
用户访问
   ↓
负载均衡 SLB
   ↓
ECS（Web服务器）×2
   ↓
RDS（MySQL数据库）
```

关键思想：

* SLB解决“访问压力”
* ECS横向扩展
* RDS提供稳定数据库服务

---

## **四、实验步骤（详细+解释）**

---

## **步骤1：创建VPC网络**

### 操作

1. 登录阿里云控制台
2. 进入「专有网络 VPC」
3. 创建VPC：

* 网段：192.168.0.0/16
* 创建交换机：192.168.1.0/24

---

### 解释

VPC相当于企业自己的“云上局域网”：

* ECS、RDS必须在同一VPC才能内网通信
* 提高安全性（避免公网暴露数据库）

---

## **步骤2：创建ECS云服务器（2台）**

### 操作

1. 进入「ECS控制台」
2. 创建实例：

* 数量：2台
* 系统：Ubuntu 20.04
* 公网IP：开启
* 安全组：开放80端口

---

### 解释

为什么要2台？

* 企业不会只用一台服务器
* 为后续“负载均衡”和“高可用”做准备

---

## **步骤3：部署Web环境（LNMP）**

在两台ECS分别执行：

```bash
apt update
apt install nginx -y
apt install mysql-client -y
apt install php php-fpm php-mysql -y
```

下载WordPress：

```bash
cd /var/www/html
wget https://wordpress.org/latest.tar.gz
tar -xvf latest.tar.gz
mv wordpress/* .
```

---

### 解释

* Nginx：Web服务器
* PHP：解析动态页面
* WordPress：模拟企业电商系统

---

## **步骤4：创建RDS数据库**

### 操作

1. 进入「RDS控制台」
2. 创建实例：

* 类型：MySQL
* 网络：选择刚才的VPC
* 开启内网访问

3. 创建数据库：

* 数据库名：wordpress
* 用户名/密码：自定义

---

### 解释

企业为什么不用本地MySQL？

* RDS具备：

  * 自动备份
  * 高可用
  * 故障切换

---

## **步骤5：连接RDS与WordPress**

修改WordPress配置文件：

```bash
cp wp-config-sample.php wp-config.php
vim wp-config.php
```

修改：

```php
define('DB_NAME', 'wordpress');
define('DB_USER', '用户名');
define('DB_PASSWORD', '密码');
define('DB_HOST', 'RDS内网地址');
```

---

### 解释

这里体现企业架构关键点：

* Web和数据库分离
* 使用内网通信（更安全、更快）

---

## **步骤6：创建负载均衡 SLB**

### 操作

1. 进入「负载均衡」
2. 创建实例
3. 添加后端服务器（2台ECS）
4. 配置监听：

* 协议：HTTP
* 端口：80

---

### 解释

SLB作用：

* 将请求分发到多台服务器
* 避免单点故障

---

## **步骤7：测试访问**

访问：

```
SLB公网IP
```

完成WordPress安装页面

---

### 测试内容

1. 多次刷新页面
2. 停止一台ECS
3. 再次访问

---

### 解释

如果访问仍正常：

说明实现了**高可用**
